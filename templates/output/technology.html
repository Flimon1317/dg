{% block teampagetitle %} COCO - Connect Online | Connect Offline {% endblock teampagetitle %}
COCO is a highly sophisticated data input system that forms the base of DigitalGreen's software stack. The creation of this system was inspired by persistent and at times debilitating issues at the field level, specifically technical issues in gathering and storing information. To alleviate these problems, DigitalGreen's software team conceived of a highly flexible and robust alternative that sought to make information gathering and input at DigitalGreen less error prone, fast, and resilient to persistent data connectivity issues in remote locations.

Most applicable to NGOs with a sizeable field operation, COCO's singular usp is the ability to take the application offline in low and limited bandwidth locations, with uninterrupted usage in the browser. Moreover, COCO's offline mode allows DigitalGreen and partner team members to access data partitioned based on their role and location and allows them to input data as if they were online. COCO is designed to support up to 100,000 users located anywhere in the world and only requires internet connectivity whenever a user is ready to synchronize their data with our global repository. Of course, users can always access COCO in online mode where inputted data updates our DigitalGreen Analytics system in real-time. Built as a robust standalone application in the Internet browser, COCO requires no additional software installation or maintenance. The system has been designed in an open-source, customizable framework that can be deployed without the need of IT/engineering staff.

Why COCO ?
COCO stands apart by addressing experiential pain points most common to NGOs with advanced field operations, and further differentiates itself from a variety of expense and feature laden commercial products. A few highlights:
  • Data gathering and analysis using technically advanced tools have escaped many NGOs primarily due to costs, access to technical resources/staff, and a clear data collection strategy.
  • Most NGOs have a decentralized operation, with a majority of field staff located in remote locations. These locations tend to have limited to no data/internet connectivity, making data input and access almost impossible.
  • Even by assuming continuous low bandwidth data/internet connectivity, archaic devices deployed in the fields still consume a lot of time, creating potential opportunity costs for a field worker.
  • While a few solutions similar to COCO exist in the realm of B2B (business to business) software, pioneered by companies such as Oracle and SalesForce, some are hard to customize, and others require maintenance and engineering staff whose high recurring costs and management time defeat the purpose of such an investment. Further exacerbating matters, most of these companies charge a per-user, per-annum license fee for input and access to data (graphs/charts) that run into thousands of dollars.

COCO Technology
COCO is architected as a Java web-framework, similar to and in proximate feature parity with popular frameworks such as Django or Ruby on Rails. The core of COCO's framework comprises of a variety of open source technologies. In light of achieving maximum speed as a browser application, COCO is architected using the GWT 2.0 (Google Web Toolkit) Java framework. To enable offline mode, COCO employs the use of Google Gears, a browser plug-in that provides a persistent Sqlite database to store data relationally and a local cache to access all requisite static content. A compiled and obfuscated Javascript file of the COCO framework is stored in the Google Gears local cache for access in offline mode. In online mode, the same compiled Javascript file and other static contents are accessed from the main DigitalGreen server. More details on our architecture should come later, but in the meantime, please contact labs@digitalgreen.org for questions.
